## Auth Pages Rubric
- Social login buttons are visible and properly styled.
- Form fields are clearly labeled with appropriate placeholders.
- Error states are visually distinct (e.g., using red or destructive colors).
- Loading states are present during form submission.
- Mobile responsiveness: No horizontal scroll at 375px width.
- Accessibility: All form fields have associated and readable labels.

## Dashboard Rubric
- Main navigation is clear, accessible, and discoverable.
- Empty states contain clear Call to Action (CTA) buttons.
- Data tables are properly formatted and sortable/filterable.
- Loading skeletons are present during initial data fetching.
- Key metrics are visually emphasized for clarity.

## Settings Page Rubric
- All settings cards feature clear labels and descriptive text.
- Save and Cancel actions are obvious and properly placed.
- Destructive actions (e.g., delete) require explicit confirmation.
- Success feedback is visible immediately after saving changes.
- Grouped settings have logical boundaries and headings.

## General UX Rubric
- No lorem ipsum or generic placeholder text is visible.
- No broken images or missing assets in the layout.
- The browser console has zero uncaught exceptions.
- All interactive elements (buttons, links) have hover/focus states.
- Color contrast is sufficient for readability throughout the page.
